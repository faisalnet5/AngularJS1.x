  *** Personal Notes (Mainly from W3School) ***   (1) While it is common in HTML applications to place scripts at the end of the element, it is recommended that you load the AngularJS library either in the or at the start of the . This is because calls to angular.module can only be compiled after the library has been loaded
(2) The AngularJS application is defined by ng-app="myApp". The ng-app directive defines the root element of an AngularJS application. It will auto-bootstrap (automatically initialize) the application when a web page is loaded
(3) The ng-controller="myCtrl" attribute is an AngularJS directive. It defines a controller
(4) The myCtrl function is a JavaScript function. AngularJS will invoke the controller with a $scope object
(5) ng-model directive binds the value of HTML controls (input, select, textarea) to application data
(6) In AngularJS, $scope is the application object, available for both the HTML (view) and the JavaScript (controller) (the owner of application variables and functions). It is very important to know which scope you are dealing with, at any given time.
(7) SCOPE: If we consider an AngularJS application to consist of:
  • View, which is the HTML.
  • Model, which is the data available for the current view.
  • Controller, which is the JavaScript function that makes/changes/removes/controls the data.
Then the scope is the Model.
(8) ROOTSCOPE: All applications have a $rootScope which is the scope created on the HTML element that contains the ng-app directive. This rootScope is available in the entire application. Remember, if a variable has the same name in both the current scope and in the rootScope, the application use the one in the current scope.
